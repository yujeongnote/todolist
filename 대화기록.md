## 1. Todo List 웹앱 초기 제작

**요청:**
> todolist를 웹사이트를 제작하려고 합니다. UI는 사각형이고 정중앙에 배치가 됩니다. 상단에 제목 본문으로 구성되며 본문 첫번째는 텍스트로 요소를 입력받습니다. 추가버튼을 누르면 일정이 추가됩니다. 일정들은 전체, 진행중, 완료 탭으로 정리가됩니다. 입력된 일정은 라디오 버튼 형태로 하단에 추가됩니다. 완료된 일정은 라디오 버튼을 통해 체크할 수 있으며 체크를 하면 남은 일정 카운터에서 차감되는 구조입니다.

**작업 결과:**
- `index.html` 파일 생성 (HTML + CSS + JavaScript 단일 파일)
- 보라색(`#4f46e5`) 헤더 + 흰색 카드 UI
- 텍스트 입력창 + "추가" 버튼 (Enter 키 지원)
- 전체 / 진행중 / 완료 탭 필터
- 라디오 버튼 형태의 할일 항목
- 완료 시 취소선 + 회색 처리
- 헤더에 남은 일정 카운터 실시간 표시
- 각 항목 우측 ✕ 삭제 버튼
- XSS 방지 (입력값 HTML 이스케이프)

**변경 사항:**

- 비선택 탭: 회색 배경(`#f1f5f9`)

**요청:**
> TODO리스트가 완성되었을 때 시간을 YYYY-MM-DD HH:mm 형식으로 나타내줘. 완성된 시간은 완료한 일에만 나타날거야.

**변경 사항:**

- 완료된 항목에만 보라색(`#a78bfa`)으로 완료 시각 표시
- 미완료로 되돌리면 완료 시각도 삭제

## 5. GitHub Push — Git 설치부터 배포까지

**요청:**
> 지금까지 한 내용을 github에 push할거야. 작업을 진행해주고, 필요한 정보가 있으면 요청해줘. git이 설치되어 있지 않다면 설치까지 진행해줘

**진행 과정:**

1. **Git 설치 확인** → 미설치 확인
2. **Git 2.54.0 설치** (`winget install --id Git.Git`)
3. **GitHub CLI 설치** (`winget install --id GitHub.cli`)
4. **GitHub 로그인 안내**
   ```
   gh auth login
   ```
5. **PAT(Personal Access Token)으로 인증 후 push**
   - 저장소: `https://github.com/yujeongnote/todolist`
   - 토큰을 URL에 삽입해 push 후 즉시 제거
6. **주의 사항:** 채팅에 토큰이 노출되었으므로 GitHub에서 재발급 권장

## 6. 설명 · 그룹 관리 · 그룹별 보기 기능 추가

**요청:**
> TODO리스트를 만들 때 설명을 넣을 수 있으면 좋겠어. 그룹 기능을 추가해서 사용자가 원하는 그룹을 만들 수 있으면 좋겠어(Home, Work 등 이름도 자유롭게 지정). TODO리스트를 만들 때 그룹을 할 수 있으면 좋겠어. TODO리스트를 그룹별로 보는 기능을 만들어줘. 각 작업을 끝낼때마다 github에 커밋해주고, 다 완료되면 push해줘.

**커밋 내역 (4개 기능 각각 분리 커밋):**

| 커밋 | 내용 |
|------|------|
| `feat: 설명 입력 기능 추가` | TODO 생성 시 선택적 설명 입력·표시 |
| `feat: 그룹 생성/삭제 관리 기능 추가` | 그룹 관리 패널(접기/펼치기), 자유 이름 입력, 삭제 시 TODO 초기화 |
| `feat: TODO 생성 시 그룹 지정 기능 추가` | 드롭다운 선택, 항목에 그룹 배지 표시 |
| `feat: 그룹별 보기 필터 추가` | 그룹 필터 + 상태 탭 동시 적용 |

**결정 사항 (대화 중 확인):**
- 그룹 필터와 상태 탭(전체/진행중/완료)을 **동시에** 적용하는 방식 채택
