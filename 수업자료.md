## 1. 웹 개발 기초 개념

웹 페이지는 세 가지 언어가 협력해서 만들어집니다.

| 역할 | 언어 | 비유 |
|------|------|------|
| 구조 (뼈대) | HTML | 건물의 기둥과 벽 |
| 디자인 (꾸미기) | CSS | 벽지, 페인트, 인테리어 |
| 동작 (기능) | JavaScript | 전기, 수도, 엘리베이터 |

### 파일 하나로 전부 작성하기

이번 프로젝트는 `index.html` 파일 하나에 HTML · CSS · JavaScript를 모두 담았습니다.

```
index.html
  ├── <style>  ← CSS 작성 영역
  ├── <body>   ← HTML 작성 영역
  └── <script> ← JavaScript 작성 영역
```

## 3. HTML — 화면 뼈대 만들기

### 기본 문서 구조

```html
<!DOCTYPE html>          <!-- 이 파일이 HTML5라고 선언 -->
<html lang="ko">         <!-- 언어: 한국어 -->
<head>
  <meta charset="UTF-8"> <!-- 한글 깨짐 방지 -->
  <title>Todo List</title>
</head>
<body>
  <!-- 화면에 보이는 내용은 여기에 -->
</body>
</html>
```

### 주요 HTML 태그

| 태그 | 용도 | 예시 |
|------|------|------|
| `<div>` | 영역 묶기 | 카드, 목록 감싸기 |
| `<input>` | 텍스트 입력 | 할일 입력창 |
| `<button>` | 버튼 | 추가 버튼 |
| `<select>` | 드롭다운 | 그룹 선택 |
| `<textarea>` | 여러 줄 입력 | 설명 입력 |
| `<span>` | 인라인 텍스트 | 배지, 라벨 |

### 이번 프로젝트의 HTML 구조

```
.card (전체 카드)
  ├── .card-header (보라색 헤더)
  │     ├── h1 "Todo List"
  │     └── 남은 일정 카운터
  └── .card-body (본문)
        ├── .input-group (입력 폼)
```

## 5. JavaScript — 기능 만들기

### 변수 선언

```javascript
let todos = [];       // 할일 목록 (배열, 바뀔 수 있음)
let groups = [];      // 그룹 목록
let currentTab = 'all'; // 현재 선택된 탭
let nextId = 1;       // 다음 할일 번호
```

### 함수 만들기

```javascript
function addTodo() {
  const text = input.value.trim(); // 입력값 가져오기 (앞뒤 공백 제거)
  if (!text) return;               // 비어있으면 중단

  todos.push({                     // 배열에 새 항목 추가
    id: nextId++,
    text: text,
    done: false
  });

  input.value = '';  // 입력창 비우기
  render();          // 화면 다시 그리기
}
```

### DOM 조작 — 화면 업데이트하기

```javascript
// HTML 요소 가져오기
const list = document.getElementById('todoList');

// innerHTML로 HTML 직접 작성
list.innerHTML = '<div>할일 내용</div>';

// 텍스트만 바꾸기
document.getElementById('counter').textContent = '남은 일정: 3개';
```

### 이벤트 처리

```javascript
// 버튼 클릭 — HTML에서 직접 연결
<button onclick="addTodo()">추가</button>

// 키보드 이벤트 — JS에서 연결
input.addEventListener('keydown', function(e) {
  if (e.key === 'Enter') addTodo(); // Enter 키를 누르면 추가
});
```

### 배열 다루기

```javascript
// 추가
todos.push({ id: 1, text: '공부하기', done: false });

// 필터링 (조건에 맞는 것만 추출)
const 미완료 = todos.filter(t => !t.done);
const 완료   = todos.filter(t => t.done);

// 특정 항목 찾기
const 항목 = todos.find(t => t.id === 3);

// 삭제 (id가 3인 것 빼고 새 배열 만들기)
todos = todos.filter(t => t.id !== 3);
```

### 템플릿 리터럴 — 동적 HTML 생성

```javascript
// 백틱(``)과 ${}를 사용해 변수를 HTML에 삽입
list.innerHTML = todos.map(t => `
  <div class="todo-item ${t.done ? 'done' : ''}">
    <span>${t.text}</span>
  </div>
`).join('');
```

## 6. 구현한 기능 상세 설명

### 6-1. 할일 추가

1. `<input>`에 텍스트 입력
2. "추가" 버튼 클릭 (또는 Enter)
3. `todos` 배열에 새 객체 push
4. `render()` 호출 → 화면 갱신

```javascript
todos.push({
  id: nextId++,        // 고유 번호
  text: '공부하기',    // 할일 내용
  desc: '2시간 동안',  // 설명 (선택)
  group: 'Study',      // 그룹 (선택)
  done: false,         // 완료 여부
  completedAt: null    // 완료 시각
});
```

### 6-2. 완료 처리 (라디오 버튼)

```javascript
function toggleTodo(id) {
  const todo = todos.find(t => t.id === id);
  todo.done = !todo.done;  // true ↔ false 토글

  if (todo.done) {
    // 완료 시각 기록 (YYYY-MM-DD HH:mm)
    const now = new Date();
    todo.completedAt = `${now.getFullYear()}-...`;
  } else {
    todo.completedAt = null;  // 미완료로 되돌리면 삭제
  }
  render();
}
```

### 6-3. 남은 일정 카운터

```javascript
const remaining = todos.filter(t => !t.done).length;
document.getElementById('counter').textContent = `남은 일정: ${remaining}개`;
```

### 6-4. 탭 필터 (전체 / 진행중 / 완료)

```javascript
const filtered = todos.filter(t => {
  if (currentTab === 'all')    return true;       // 전체
  if (currentTab === 'active') return !t.done;    // 진행중
  if (currentTab === 'done')   return t.done;     // 완료
});
```

### 6-5. 그룹 관리

```javascript
// 그룹 추가
function addGroup() {
  const name = document.getElementById('groupInput').value.trim();
  if (!name || groups.includes(name)) return; // 빈값 · 중복 방지
  groups.push(name);
  renderGroups(); // 그룹 목록 갱신
}

// 그룹 삭제 → 해당 그룹의 할일은 '그룹 없음'으로 초기화
function deleteGroup(name) {
  groups = groups.filter(g => g !== name);
  todos.forEach(t => { if (t.group === name) t.group = ''; });
  renderGroups();
  render();
}
```

### 6-6. 그룹 + 상태 동시 필터

```javascript
const filtered = todos.filter(t => {
  const statusOk = currentTab === 'all' || (currentTab === 'active' ? !t.done : t.done);
  const groupOk  = currentGroup === '' || t.group === currentGroup;
  return statusOk && groupOk;  // 둘 다 만족해야 표시
});
```

### 6-7. XSS 방지 (보안)

사용자가 입력한 텍스트를 HTML로 직접 삽입하면 악성 스크립트가 실행될 수 있습니다.  
이를 막기 위해 특수문자를 변환합니다.

```javascript
function escapeHtml(str) {
  return String(str)
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')    // < → &lt; (스크립트 태그 방지)
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;');
}

// 사용 예
`<span>${escapeHtml(t.text)}</span>`
```

## 7. Git — 버전 관리 도구

### Git이란?

코드의 변경 이력을 저장하고 관리하는 도구입니다.  
마치 게임의 **세이브 포인트**처럼, 원하는 시점으로 되돌아갈 수 있습니다.

```
[작업] → [저장(commit)] → [작업] → [저장(commit)] → ...
                                        ↑ 언제든 이 시점으로 돌아올 수 있음
```

### 설치

Windows에서 winget으로 설치:

```powershell
winget install --id Git.Git -e --source winget
```

설치 확인:

```bash
git --version
# 출력: git version 2.54.0.windows.1
```

### 기본 명령어

#### 저장소 시작

```bash
git init          # 현재 폴더를 Git 저장소로 초기화
```

#### 사용자 정보 설정 (최초 1회)

```bash
git config user.name "홍길동"
git config user.email "hong@example.com"
```

#### 변경사항 저장하기

```bash
git add index.html        # 특정 파일 스테이징
git add .                 # 모든 변경 파일 스테이징

git commit -m "메시지"    # 스테이징된 내용 저장(커밋)
```

> **스테이징(Staging)이란?**  
> 사진 찍기 전 포즈를 잡는 것처럼, 커밋할 파일을 미리 선택하는 단계입니다.

#### 커밋 흐름 이해하기

```
작업 공간 → (git add) → 스테이지 → (git commit) → 저장소
(Working)               (Stage)                   (Repository)
```

#### 이력 확인

```bash
git log --oneline
# 출력 예시:
# a8ad90a feat: 스마트폰 프레임 제거
# 57d5993 docs: README 추가
# 1f689ef feat: 그룹별 보기 필터 추가
```

### 브랜치 — 독립적인 작업 공간

브랜치는 **나뭇가지**처럼 메인 코드에서 분기해 독립적으로 작업하는 공간입니다.

```
main ──────────────────────────────→ (안정적인 코드)
          │
          └── feat/new-feature ────→ (새 기능 개발)
                    │
                    └── (완성되면 main으로 합치기 = merge)
```

```bash
git branch                      # 브랜치 목록 보기
git checkout -b feat/새기능     # 새 브랜치 만들고 이동
git checkout main               # main 브랜치로 이동
git merge feat/새기능           # 현재 브랜치에 합치기
```

### 커밋 메시지 규칙 (Conventional Commits)

```
feat: 새 기능 추가
fix: 버그 수정
docs: 문서 수정
style: 코드 스타일 변경 (기능 변화 없음)
refactor: 코드 리팩터링
```

예시:
```
feat: 그룹별 보기 필터 추가
fix: 카드 너비 축소 및 수직 중앙 배치
docs: README 추가 및 데이터 저장 방식 안내
```

## 8. GitHub — 코드 저장소

### GitHub이란?

Git으로 관리하는 코드를 **인터넷에 올려두는 창고**입니다.  
- 코드 백업
- 다른 사람과 협업
- 포트폴리오 공개

### GitHub CLI 설치

터미널에서 GitHub에 로그인할 수 있게 해주는 도구입니다.

```powershell
winget install --id GitHub.cli -e --source winget
```

### 로그인

```bash
gh auth login
# 1. GitHub.com 선택
# 2. HTTPS 선택
# 3. Login with a web browser 선택
# 4. 브라우저에서 코드 입력 후 인증
```

### 원격 저장소 연결하기

```bash
# GitHub에서 저장소(Repository)를 먼저 만든 후:
git remote add origin https://github.com/사용자명/저장소명.git

# 브랜치 이름을 main으로 설정
git branch -M main

# 처음 올릴 때
git push -u origin main

# 이후부터는
git push
```

### 전체 작업 흐름 요약

```
1. GitHub에서 저장소 생성
2. git init (로컬 폴더 초기화)
3. 코드 작성
4. git add .
5. git commit -m "메시지"
6. git remote add origin [URL]
7. git push origin main
```

### 브랜치를 활용한 협업 흐름

```
1. git checkout -b feat/기능명   # 새 브랜치 생성
2. 코드 작성 + 커밋
3. git push origin feat/기능명  # 브랜치 올리기
4. GitHub에서 Pull Request 생성
5. 코드 리뷰
6. main에 merge
```

## 9. Vercel — 웹사이트 배포

### Vercel이란?

작성한 웹 페이지를 **전 세계 누구나 접속할 수 있는 주소**로 공개해주는 서비스입니다.

- GitHub 저장소와 연결하면 **코드를 push할 때마다 자동으로 배포**
- 무료 플랜으로도 충분히 사용 가능
- `https://프로젝트명.vercel.app` 형태의 주소 제공

### 배포 방법 (GitHub 연동)

**① Vercel 가입**
1. [vercel.com](https://vercel.com) 접속
2. "Sign Up" → "Continue with GitHub" 클릭
3. GitHub 계정으로 로그인

**② 프로젝트 가져오기**
1. Vercel 대시보드에서 "Add New → Project" 클릭
2. GitHub 저장소 목록에서 `todolist` 선택
3. "Import" 클릭

**③ 배포 설정**

| 항목 | 설정값 |
|------|--------|
| Framework Preset | Other |
| Root Directory | ./ (기본값) |
| Build Command | (비워두기) |
| Output Directory | ./ (기본값) |

4. "Deploy" 클릭 → 1~2분 후 배포 완료

**④ 결과**

```
배포 완료 주소: https://todolist-사용자명.vercel.app
```

### 자동 배포 흐름

```
코드 수정 → git push origin main → Vercel이 자동으로 감지 → 새 버전 배포
```

### Vercel CLI로 배포 (터미널 방식)

```bash
# Vercel CLI 설치
npm install -g vercel

# 배포
vercel

# 프로덕션 배포
vercel --prod
```

#### 과제 5 — 할일 수정 기능
완료되지 않은 할일을 클릭하면 텍스트를 수정할 수 있게 만들어보세요.

```javascript
// 힌트: 할일 텍스트를 클릭하면 input으로 바뀌는 기능
function editTodo(id) {
  const todo = todos.find(t => t.id === id);
  const newText = prompt('수정할 내용을 입력하세요:', todo.text);
  if (newText && newText.trim()) {
    todo.text = newText.trim();
    render();
  }
}
```

#### 과제 6 — 로컬 스토리지 저장
페이지를 새로고침해도 데이터가 유지되도록 `localStorage`를 사용해보세요.

```javascript
// 힌트: 데이터 저장
function save() {
  localStorage.setItem('todos', JSON.stringify(todos));
  localStorage.setItem('groups', JSON.stringify(groups));
}

// 힌트: 데이터 불러오기 (페이지 로드 시)
const saved = localStorage.getItem('todos');
if (saved) todos = JSON.parse(saved);

// render() 이후마다 save() 호출
```
